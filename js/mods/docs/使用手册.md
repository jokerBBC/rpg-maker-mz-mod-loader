## 一、游戏制作者

### 1.1 将 ModLoader 接入自己的游戏

ModLoader 以 `js/mods/ModLoader.js` 形式分发，推荐 **注入模式**（插件模式亦可，但需注意加载时序）。

**注入模式（推荐）**：修改游戏根目录 `index.html`，在 `main.js` **之前**插入：

```html
<script type="text/javascript" src="js/mods/ModLoader.js"></script>
```

**插件模式**：在 RMMZ 插件管理器中添加 `ModLoader.js`。

随游戏更新时，一并推送 `js/mods/` 目录（含 `ModLoader.js`、`config/`、`docs/` 等）。玩家 Mod 配置保存在 `mod_config.json`，**不写入** `plugins.js`，官方插件更新后 Mod 开关不会丢失。

### 1.2 开启 Steam 创意工坊

编辑 `js/mods/config/modloader_config.json` 的 `workshop` 段：

```json
{
  "ml_theme": "dark",
  "ml_language": "zh_CN",
  "workshop": {
    "enabled": true,
    "steamAppId": "你的游戏AppID",
    "steamLibraryPath": ""
  }
}
```

| 字段 | 说明 |
| --- | --- |
| `enabled` | `true`：扫描并加载工坊 Mod；`false`：保留「创意工坊」筛选入口，列表提示「游戏未开启创意工坊功能」，不扫描 |
| `steamAppId` | **必改**为你的 Steam AppID（与 `steam_appid.txt`、Steamworks 工坊后台一致）。仓库示例 `4379740` 仅用于本仓库联调 |
| `steamLibraryPath` | 一般留空；Steam 装在非默认库盘或多库时填**库根**，如 `D:/SteamLibrary` |

> **运行环境**：V3.16.1 起 ModLoader 依赖 Steam 正版安装路径解析工坊目录。非正版环境无法使用工坊功能。

开通工坊后，将 `enabled` 设为 `true` 并填入正确 AppID，**随游戏更新推送给玩家**即可。

### 1.3 开启创意工坊后如何自测

1. 确认 `workshop.enabled: true`，AppID 正确，游戏为 Steam 正版启动  
2. F5 进游戏 → 打开 Mod 管理器 → 左下角版本应为 **V4.1.1**（或当前发布版）  
3. 筛选 **创意工坊**，应能看到已订阅的测试包  
4. 点击 **刷新工坊**，列表应更新；`_workshop/<fileId>/` junction 应重建  
5. 开启测试 Mod → **保存** → F5，确认游戏正常启动且 Mod 生效  

**本仓库联调自测包**（AppID `4379740`）：

```
<Steam库>/steamapps/workshop/content/4379740/
  3000000001/   # 单脚本 + manifest + preview
  3000000002/   # 依赖 3000000001（@base）
  3000000003/   # 无 modloader.json，单脚本
  3000000004/   # 多脚本包（3 行列表）
```

各包脚本、`preview.png`、`modloader.json` 均在 **包根目录**（V4.1 扁平布局，无 `js/mods/` 子目录）。详细测试项见 [`V4.1_测试文档.md`](V4.1_测试文档.md)。

**发布前检查**：

- 工坊包结构与本地 `_localmods` 包结构一致（见 [3.2 包结构规范](#32-包结构规范)）  
- `modloader.json` 的 `entries` 仅写文件名，禁止路径  
- 详情区 preview 可显示；缺失时显示「无预览图」  

### 1.4 管理器语言包如何拓展

管理器 UI 文案位于 `js/mods/config/language/`：

```
config/language/
├── zh_CN.json
├── zh_TW.json
└── en.json
```

**添加新语言**：

1. 复制 `zh_CN.json` 为 `ja.json`（示例）  
2. 修改文件头元数据：
   ```json
   {
     "_langCode": "ja",
     "_langName": "日本語",
     ...
   }
   ```
3. 翻译其余键值（键名不可改，仅改值）  
4. F5 重开管理器 → ⚙ 系统设置 → 语言下拉应出现新项  

**翻译查找链**：当前语言 → `zh_CN` → 键名原文（不崩溃）。

Mod 插件内的 `@text` 参数别名由 Mod 作者自行定义，与管理器语言包无关。

## 二、玩家

### 2.1 基本使用

1. 进入游戏，点击左上角 **模组管理器**  
2. 用 **全部 / 本地 / 创意工坊** 筛选列表；工坊开启时可点 **刷新工坊**  
3. 选中 Mod，右侧查看详情（来源、依赖、参数等）  
4. 开关 Mod；有 ⚙ 的可改参数；改完点 **保存**  
5. **F5 刷新游戏**，开关与参数才会生效  

> 修改 Mod 开关、参数或排序后必须 F5。工坊 Mod 请在 **Steam 客户端** 订阅/取消订阅；游戏内仅开关、排序与改参。

### 2.2 本地 Mod 与创意工坊 Mod

| 类型 | 来源 | 能否删除 | 能否拖放安装覆盖 |
| --- | --- | --- | --- |
| **本地** | `js/mods/_localmods/<包名>/` | 可以（整包删除） | 可以 |
| **创意工坊** | Steam 订阅，经 `_workshop/` 桥接加载 | 不可以 | 不可以 |

工坊 Mod 在列表中带 **工坊** 角标；删除模式下工坊行 **不显示** 垃圾箱。工坊 Mod **可排序**（需在「全部」筛选下）。

### 2.3 安装本地 Mod

打开管理器 → **安装 Mod** 入口：

| 方式 | 说明 |
| --- | --- |
| 拖放 `.js` | 自动安装到 `_localmods/<基名>/<基名>.js` |
| 浏览选 `.js` | 同上，可多选 |
| 拖放整个 `mods` 文件夹 | 合并复制到 `js/mods/` 根（含 `_localmods`、config 等） |

无效拖放（非 `.js` 且非 mods 文件夹）会提示：**拖放安装仅支持：.js 文件或 mods 文件夹！**

### 2.4 配置保存在哪

| 文件 | 内容 |
| --- | --- |
| `js/mods/mod_config.json` | 各 Mod 开关、参数、排序（唯一配置源） |
| `js/mods/config/modloader_config.json` | 管理器主题、语言、工坊开关 |

**V4.1.1 起**：若你从旧版升级，`mod_config.json` 里可能仍是 `../mods/<名>` 旧键。打开管理器后 **保存一次**，会自动改写为 `local:<包名>:<脚本基名>` 新键，无需手动改 JSON。

### 2.5 从旧版整合包升级（V4.1）

1. 备份 `js/mods/config/` 与 `js/mods/mod_config.json`  
2. 删除 `js/mods/` 下除 `config/` 和 `mod_config.json` 外的内容  
3. 覆盖作者提供的新版 `mods` 文件夹（已含 `_localmods/` 及修复后的资源路径）  
4. F5 进游戏 → 打开管理器确认开关/参数 → **保存** → 再 F5  

Mod 脚本与资源路径需使用新版包结构；仅保留旧 `mod_config.json` 不够，必须搭配新版 Mod 文件。

## 三、Mod 作者

### 3.1 元数据规范

Mod 为 RMMZ 插件格式，文件头注释需包含：

```javascript
/*:
 * @target MZ
 * @plugindesc 我的 Mod 简介
 * @author 作者名
 * @version V1.0.0
 *
 * @param myOption
 * @text 选项显示名
 * @type number
 * @default 1
 * @min 0
 * @max 10
 *
 * @help
 * 帮助文本（支持 Markdown 风格换行）
 */
```

**常用标签**：

| 标签 | 说明 |
| --- | --- |
| `@text` | 参数在管理器中的中文显示名 |
| `@base` | 前置依赖（空格分隔多个插件名，可带或不带 `.js`） |
| `@orderAfter` | 应排在某插件之后 |
| `@orderBefore` | 应排在某插件之前 |
| `@define-schema` / `@schema` | struct / table 模板定义 |

**依赖检测**：管理器先查游戏原生插件，再查列表内全部 Mod（含工坊）。游戏原生插件已开启则顺序天然满足。

**参数类型**：`number`、`boolean`、`string`、`select`、`color`、`note` / `multiline_string`、数据库引用（`actor`/`skill`/…）、`struct`、`table`。所有 `@param` 必须提供 `@default`，否则参数编辑会被锁定。

**struct / table 示例**：见 `_localmods/TestSchemaMod/TestSchemaMod.js`、`_localmods/mydrop/mydrop.js`。

### 3.2 包结构规范

V4.1 起，**本地**与**工坊**使用同一套包根布局：

**单脚本包**（文件夹名 = 脚本基名）：

```
js/mods/_localmods/老板键模式/
  老板键模式.js
  preview.png          # 可选
```

**多脚本包**（文件夹名 = 包名，列表每行一个 `.js`）：

```
js/mods/_localmods/多脚本包名/
  modloader.json       # 可选
  preview.png          # 可选，包内共用
  Core.js
  FeatA.js
```

**Steam 工坊包**（结构相同，路径在 Steam 库）：

```
<publishedFileId>/
  modloader.json
  preview.png
  YourMod.js
```

| 规则 | 说明 |
| --- | --- |
| 脚本位置 | 包根一层 `*.js`，**不递归**子目录 |
| 无 `modloader.json` | 自动扫描包根全部 `.js`（跳过 `ModLoader.js`） |
| `modloader.json` | `entries` **仅写文件名**，禁止 `/`、`\`、`..` |
| `preview.png` | 固定文件名，包根；详情缩略 + 点击 NW 弹窗大图 |
| 资源文件 | 放在包内子目录（如 `img/`、`docs/`），脚本内路径写相对于游戏根或包内路径 |

**mod_config 键名**：

| 来源 | 键名 | loadPath |
| --- | --- | --- |
| 本地 | `local:<包名>:<脚本基名>` | `../mods/_localmods/<包名>/<脚本基名>` |
| 工坊 | `ws:<fileId>:<脚本基名>` | `../mods/_workshop/<fileId>/<脚本基名>` |

**已废弃（V4.1 不再支持）**：

- 本地 `js/mods/*.js` 扁平放置  
- 工坊包内 `js/mods/` 子目录  
- `entries` 中带路径  

### 3.3 管理器语言包拓展

若你为 **ModLoader 本身** 贡献 UI 翻译（非 Mod 插件内文案），步骤同 [1.4 管理器语言包如何拓展](#14-管理器语言包如何拓展)。Mod 插件的 `@text` 由作者在插件头自行编写，不走 `config/language/`。

### 3.4 制作 Mod 时参考示例

| 示例 | 路径 | 说明 |
| --- | --- | --- |
| `TestMod` | `_localmods/TestMod/TestMod.js` | 基础参数类型、颜色、数据库引用 |
| `TestSchemaMod` | `_localmods/TestSchemaMod/` | struct、table、嵌套 schema、@text |
| `mydrop` | `_localmods/mydrop/mydrop.js` | table 配置怪物掉落 |
| 工坊 3000000001~4 | `steamapps/workshop/content/4379740/` | 单脚本 / 依赖 / 多脚本 / preview 尺寸 |

**资源路径提示**：V4.1 后 Mod 资源应放在包目录内，例如：

```
js/mods/_localmods/攻略查看器/
  攻略查看器.js
  img/攻略图标.png
  docs/guide/...
```

脚本内加载：`js/mods/_localmods/攻略查看器/img/攻略图标.png`（NW.js 路径相对于游戏根目录）。
