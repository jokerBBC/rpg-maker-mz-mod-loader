## 1. 目标目录结构

```
js/mods/
  ModLoader.js
  mod_config.json
  config/ docs/ libs/ tools/ img/ …
  _workshop/<fileId>/          # junction（自动生成）
  _localmods/
    老板键模式/
      老板键模式.js
      preview.png
    多脚本包名/
      modloader.json
      preview.png
      Core.js

<publishedFileId>/             # 工坊订阅包（同布局）
  modloader.json
  preview.png
  Core.js
```

---

## 2. mod_config 键名


| 来源  | 配置键                  | loadPath                            |
| --- | -------------------- | ----------------------------------- |
| 本地  | `local:<包名>:<脚本基名>`  | `../mods/_localmods/<包名>/<脚本基名>`    |
| 工坊  | `ws:<fileId>:<脚本基名>` | `../mods/_workshop/<fileId>/<脚本基名>` |

## 3. 代码改造清单

### Phase 1 — 共用包发现

- `discoverPackageScripts(packageRoot)`
- `resolvePackageEntryFileName`（禁止路径）
- 工坊改用包根 flatten

### Phase 2 — 扫描与桥接

- `LOCALMODS_DIR`
- `scanLocalMods` 仅扫 `_localmods`
- 删除根目录 `*.js` 扫描
- `syncWorkshopBridge` junction 包根
- `modLookup` 注册 `local:`

### Phase 3 — 安装

- 单/多 `.js` → `_localmods/<基名>/<基名>.js`
- 不支持拖文件夹当包
- mods 文件夹全量复制到 `js/mods/` 根

### Phase 4 — 删除

- 整包删除 `_localmods/<包名>/`
- 多脚本包确认框整包提示 + 列出脚本
- 删除后 `reassignOrders` + `persistModListToConfig`

### Phase 5 — 预览

- 本地 Mod 复用工坊 preview UI

### Phase 6 — 文档与自测

- 作者规范、CHANGELOG
- Steam 自测包 flatten
- V4.1.0
- `tools/migrate-*.js`

## 4. 仓库外脚本（不内置 ModLoader）


| 脚本                                         | 作用                            |
| ------------------------------------------ | ----------------------------- |
| `tools/migrate-local-mods-to-localmods.js` | 根 `*.js` → `_localmods/<基名>/` |
| `tools/migrate-mod-config-keys.js`         | `../mods/X` → `local:X:X`     |


---

## 5. 明确不做

- 双轨扫描
- 管理器内 **文件/目录** 自动迁移（仍用外置 `migrate-*.js`）
- 拖入文件夹当 Mod 包安装
- 多脚本包按行逐个删
- 本次批量迁移 mod 附带资源

> **V4.1.1 起**：管理器 **读取** 时兼容 V3.x `../mods/<脚本基名>` 配置键（含 V1 布尔值格式）；**保存** 时全量写入 `local:` / `ws:` 新键，玩家无需跑 `migrate-mod-config-keys.js`。

---

## 6. 作者自用迁移步骤

```powershell
cd <游戏根目录>
node js/mods/tools/migrate-local-mods-to-localmods.js --dry-run
node js/mods/tools/migrate-local-mods-to-localmods.js
# mod_config 键：V4.1.1+ 打开管理器保存一次即可；或仍可用：
# node js/mods/tools/migrate-mod-config-keys.js --backup
```

然后 F5 刷新，逐个 Mod 修复脚本内 `js/mods/img/` 等资源路径。
