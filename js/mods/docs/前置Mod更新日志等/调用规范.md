# Mod 作者调用规范（数据 + 资源）

> 版本：V2.1 | 适用：ModLoader V4.1.x + ModDataLoader V2.x + ModResourceLoader V2.x

---

## 1. 声明前置依赖

在你的插件头部 JSDoc 中使用 `@base` 声明依赖：

```
/*:
 * @target MZ
 * @plugindesc [功能] 你的 Mod 描述
 * @base ModDataLoader
 * @orderAfter ModDataLoader
 *
 * @help
 * ...
 */
```

- `@base ModDataLoader` — 告诉 ModLoader 本插件依赖 ModDataLoader，UI 会显示依赖状态警告
- `@orderAfter ModDataLoader` — 确保加载顺序在 ModDataLoader 之后

> ⚠ **重要提醒**：ModLoader 目前对缺失 `@base` 只显示 UI 警告，**不会阻止 JS 执行**。
> 如果玩家关闭了 ModDataLoader 但你的 Mod 仍然启用，你的 JS 代码仍会被执行。
> 因此 **必须在代码中做前置存在性检查**。

---

## 2. 入口前置检查（铁律）

在你的 IIFE 最顶部加入存在性检查。如果 ModDataLoader 不存在，立即 `return` 退出，
**不安装任何 hook、不调用任何 API**：

```javascript
(() => {
    'use strict';

    const PLUGIN = '你的Mod名';

    // ═══ 前置依赖检查 ═══
    if (!window.ModDataLoader) {
        console.warn(`[${PLUGIN}] ModDataLoader 未启用，本 Mod 自动跳过`);
        return;  // 静默退出，不安装任何 hook
    }

    // ... 以下所有代码只在 ModDataLoader 存在时执行 ...
})();
```

这是**最重要的规则**。没有这个检查，任何对 `ModDataLoader.xxx` 的调用都会抛出
`ReferenceError`，可能导致游戏崩溃且玩家无法自行修复。

## 3. Hook 安全规则

即使入口检查通过，在以下场景中仍建议做防御性检查：

### 3.1 DataManager.onLoad 覆写

```javascript
const _orig_onLoad = DataManager.onLoad;
DataManager.onLoad = function(object) {
    _orig_onLoad.call(this, object);

    // 防御性检查：确保 ModDataLoader 仍然可用
    if (!window.ModDataLoader) return;

    if (object === $dataItems) {
        const report = ModDataLoader.getConflictReport();
        // ...
    }
};
```

### 3.2 Scene 覆写

```javascript
const _Scene_Map_start = Scene_Map.prototype.start;
Scene_Map.prototype.start = function() {
    _Scene_Map_start.call(this);

    if (!window.ModDataLoader) return;

    const id = ModDataLoader.findIdByStableKey('Items', 'your-mod:item-key');
    // ...
};
```

### 3.3 定时器 / 延迟回调

```javascript
setTimeout(() => {
    if (!window.ModDataLoader) return;
    ModDataLoader.registerData('Items', [...], 'merge');
}, 1000);
```

## 4. 常用 API 速查

### 数据注册

```javascript
// 合并模式：只修改指定字段，保留其他字段
ModDataLoader.registerData('Items', [
    { id: 1, name: '新名称', price: 999 }
], 'merge');

// 替换模式：整个条目替换
ModDataLoader.registerData('Items', [
    { id: 2, name: '完全替换的物品', ... }
], 'replace');

// 新增模式：只在不存在的字段上生效
ModDataLoader.registerData('Items', [
    { id: 3, name: '新字段值' }
], 'add');
```

### 稳定标识新增（推荐用于新增条目）

```javascript
ModDataLoader.registerStableEntry('Items', {
    id: 9999,  // 占位 ID，会被自动收紧
    stableKey: 'your-mod-prefix:item-name',
    name: '新增物品',
    iconIndex: 176,
    // ... 其他 RMMZ 标准字段 ...
});
```

> stableKey 格式建议：`mod名:物品描述`，用冒号分隔，全小写，连字符分隔单词。
> 这样能避免不同 Mod 之间的 stableKey 冲突。

### 地图数据

```javascript
ModDataLoader.registerMapData(1, {  // Map001
    events: [
        { id: 1, name: '修改后的事件名', pages: [...] }
    ]
}, 'merge');
```

### 诊断命令（F12 控制台）

```javascript
ModDataLoader.getDataRegistry()     // 查看所有数据注册
ModDataLoader.getConflictReport()   // 查看冲突报告
ModDataLoader.getMigrationLog()     // 查看 stableKey 迁移记录
```

## 5. modloader.json 清单格式

如果你的 Mod 包需要声明数据文件自动加载：

```json
{
    "schemaVersion": 1,
    "title": "你的 Mod 名称",
    "entries": ["YourMod.js"],
    "data": {
        "records": [
            {
                "file": "data/YourItems.json",
                "type": "items",
                "format": "array"
            }
        ],
        "patches": [
            {
                "target": "Items",
                "index": 5,
                "merge": { "name": "补丁改名" }
            }
        ]
    }
}
```

- `data.records` — 自动加载的 JSON 数据文件（`type` 对应 `$dataXxx` 去掉 `$data` 后的名称）
- `data.patches` — 字段级补丁（`index` 用于数组表，省略用于对象表如 System）
- `format: "array"` — 文件是 RMMZ 标准数组格式 `[{id:1,...}, ...]`
- `format: "object"` — 文件是对象格式 `{key: value, ...}`

## 6. 完整模板

```javascript
/*:
 * @target MZ
 * @plugindesc [功能] 你的 Mod 描述
 * @author 你的名字
 * @version 1.0.0
 * @base ModDataLoader
 * @orderAfter ModDataLoader
 *
 * @help
 * 你的 Mod 帮助文本
 */

(() => {
    'use strict';

    const PLUGIN = 'YourModName';

    // ═══ 前置依赖检查（铁律，不可省略）═══
    if (!window.ModDataLoader) {
        console.warn(`[${PLUGIN}] ModDataLoader 未启用，本 Mod 自动跳过`);
        return;
    }

    const MDL = window.ModDataLoader;
    const params = Object.assign(
        {},
        PluginManager.parameters(PLUGIN),
        PluginManager.parameters(`../mods/_localmods/${PLUGIN}/${PLUGIN}`)
    );

    // ── 注册数据修改 ──
    MDL.registerData('Items', [
        { id: 1, name: '修改后的物品名', price: 500 }
    ], 'merge');

    // ── 新增条目（带稳定标识）──
    MDL.registerStableEntry('Items', {
        id: 9999,
        stableKey: `${PLUGIN.toLowerCase()}:custom-item`,
        name: '自定义物品',
        iconIndex: 176,
        itypeId: 1,
        price: 100,
        consumable: true,
        occasion: 0,
        animationId: 0,
        hitType: 0,
        damage: { type: 0, elementId: 0, formula: '0', variance: 0, critical: false },
        effects: [],
        params: [0, 0, 0, 0, 0, 0, 0, 0],
        repeats: 1, scope: 0, speed: 0, successRate: 100, tpGain: 0,
        description: '',
        note: ''
    });

    // ── Hook（带防御性检查）──
    const _orig_onLoad = DataManager.onLoad;
    DataManager.onLoad = function(object) {
        _orig_onLoad.call(this, object);
        if (!window.ModDataLoader) return;

        if (object === $dataItems) {
            console.log(`[${PLUGIN}] 数据已加载`);
        }
    };

    console.log(`[${PLUGIN}] loaded`);
})();
```

## 7. ModResourceLoader 资源系统

### 7.1 声明依赖

```
 * @base ModResourceLoader
 * @orderAfter ModResourceLoader
```

入口检查：
```javascript
if (!window.ModResourceLoader) {
    console.warn(`[${PLUGIN}] ModResourceLoader 未启用，本 Mod 自动跳过`);
    return;
}
```

### 7.2 替换原游戏资源（manifest 声明，零 JS 代码）

在 `modloader.json` 中添加 `modId` 和 `resources` 字段：

```json
{
    "modId": "MyMod",
    "title": "MyMod",
    "entries": ["MyMod.js"],
    "resources": {
        "img/菜单/现实时间图标": "img/clock.png",
        "img/pictures/Actor1": "img/myActor1.png"
    }
}
```

- `modId`：Mod 作者自定义的稳定标识，用于 `loadBitmap` API。**必填**，创意工坊上传后包名会变成工坊 ID，但 modId 不变。
- `resources`：键为原游戏资源路径，值为 Mod 包内的相对路径。

ModResourceLoader 扫描 manifest 时自动注册替换。**Mod 的 JS 文件无需任何替换代码。**
Mod 作者不需要知道自己的安装路径——系统自动解析。

### 7.3 新增资源（一行 API）

当 Mod 需要加载游戏原本不存在的图片时：

```javascript
const bitmap = ModResourceLoader.loadBitmap('MyMod', 'img/myIcon');
```

- 第一个参数：Mod 包名（与 `mod_config.json` 中的 packageName 一致）
- 第二个参数：相对于 Mod 根目录的路径（不含 `.png`）

典型用法——在场景中显示自定义图标：
```javascript
const sprite = new Sprite();
sprite.bitmap = ModResourceLoader.loadBitmap('MyMod', 'img/indicator');
sprite.x = 16;
sprite.y = Graphics.height - 64;
this.addChild(sprite);
```

### 7.4 路径解析原理

```
Mod 包结构:
  _localmods/MyMod/
  ├── MyMod.js
  ├── modloader.json        ← resources 声明在这里
  └── img/
      ├── clock.png         ← 替换用（manifest 声明）
      └── indicator.png     ← 新增用（loadBitmap 加载）

创意工坊 Mod:
  _workshop/2938471023/
  ├── MyMod.js
  ├── modloader.json        ← 同样的声明
  └── img/
      └── clock.png         ← 系统自动解析到工坊目录
```

ModResourceLoader 自己扫描每个 mod 的 manifest，知道每个 mod 的目录位置。
**Mod 作者始终用相对路径，代码在本地和创意工坊通用。**

## 8. 常见错误与预防

| 错误 | 原因 | 后果 | 预防 |
|------|------|------|------|
| `ReferenceError: ModDataLoader is not defined` | 未做前置检查 | 游戏崩溃，玩家无法自行修复 | IIFE 入口 `return` |
| `TypeError: Cannot read property 'registerData' of undefined` | 在 hook 中调用但未检查 | 功能失效或崩溃 | hook 内 `if (!window.ModDataLoader) return` |
| 数据不生效 | `@base` 声明但 mod_config.json 中顺序错误 | 数据注册被忽略 | 同时使用 `@base` + `@orderAfter` |
| stableKey 冲突 | 多个 Mod 使用相同 stableKey | ID 迁移混乱 | 使用 `mod名:描述` 格式确保唯一 |
| 图片替换不生效 | modloader.json 中 resources 路径写错 | 原图未被替换 | 键为原游戏路径（不含 .png），值为 mod 内相对路径（含 .png） |
| loadBitmap 返回空白 | modName 与 mod_config.json 中包名不一致 | 找不到 mod 路径 | 确认 modName 拼写与配置一致 |
